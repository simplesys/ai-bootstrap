# Локальные навыки проекта

Этот файл описывает повторяемые проектные задачи. Используйте эти инструкции только в рамках данного репозитория.

## `updocs`: синхронизация документации между `lang/ru` и английскими файлами

Короткое название: `updocs`.

Варианты вызова:

- `updocs` - обновить англоязычные файлы на основе `lang/ru`;
- `updocs --en` - обновить файлы в `lang/ru` на основе соответствующих англоязычных файлов.

Используйте этот навык, когда пользователь просит актуализировать перевод, синхронизировать документацию или обновить файлы одной языковой версии на основе другой.

Цель: без флага считать файлы в `lang/ru` источником истины и обновлять соответствующие англоязычные файлы. С флагом `--en` считать источником истины англоязычные файлы и обновлять соответствующие файлы в `lang/ru`.

Порядок работы:

1. Определите режим работы:
   - без флага `--en` источник - `lang/ru`, цель - соответствующие англоязычные файлы;
   - с флагом `--en` источник - англоязычные файлы, цель - соответствующие файлы в `lang/ru`.
2. Для режима без `--en` найдите все актуальные файлы в `lang/ru`.
3. Определите соответствующий английский путь:
   - `lang/ru/README.md` -> `README.md`;
   - `lang/ru/AGENTS.md` -> `AGENTS.md`;
   - `lang/ru/<path>` -> `<path>`, например `lang/ru/examples/README.md` -> `examples/README.md`.
4. Для режима `--en` найдите все актуальные англоязычные файлы, для которых предусмотрено прямое соответствие в `lang/ru`, и примените обратное сопоставление:
   - `README.md` -> `lang/ru/README.md`;
   - `AGENTS.md` -> `lang/ru/AGENTS.md`;
   - `<path>` -> `lang/ru/<path>`, например `examples/README.md` -> `lang/ru/examples/README.md`.
5. Если соответствующий целевой файл отсутствует, создайте его.
6. Переведите содержимое на язык целевого файла, сохраняя Markdown-структуру, списки, ссылки, пути и смысл.
7. Не удаляйте и не изменяйте исходные файлы выбранного режима, если пользователь прямо не просит об этом.
8. Не трогайте чужие изменения вне файлов, которые являются прямыми соответствиями исходных файлов выбранного режима.
9. После правок без `--en` проверьте, что вне `lang/ru` не осталось случайной кириллицы:

   ```bash
   rg -n '\p{Cyrillic}' --glob '!lang/ru/**' .
   ```

10. В итоговом ответе перечислите обновленные файлы и укажите, что тесты не запускались, если изменения были только в документации.
