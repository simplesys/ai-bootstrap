## CodeGraph

1. При работе с кодом: поиске файлов, кода, любом анализе проекта, диагностике ошибок задействуй MCP CodeGraph.

2. Если данные выглядят неактуальными, обратись к `codegraph_status`. Не запускай `codegraph sync` если активен watcher.

3. Ознакомление с кодом начинай с `codegraph_explore`.
Команду `codegraph_search` - используй только для поиска имени символа.
Команду `codegraph_node` - для извлечения полного тела символа.
Команды `codegraph_callers` и `codegraph_callees` - для анализа связей в коде.
Команду `codegraph_impact` - перед проведением рефакторинга.

4. Не открывай повторно файлы, чей исходный код CodeGraph уже нашёл, кроме ситуаций, когда требуется редактирование, проверка незакоммиченных изменений или получено предупреждение об устаревшем индексе. Применяй команы `rg`, `sed` и прямое чтение файлов лишь для неподдерживаемых индексом форматов и случаев, точного текстового поиска или в остальных случаях, когда возможностей CodeGraph оказывается недостаточно.
